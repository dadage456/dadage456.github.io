搭建APNs推送服务

#搭建APNs推送服务(三)：服务器与APNs连接

与APNs通信有两种方式

  • 验证令牌与 APNs 通信

    只需要配置一个密钥标识符，可以供多个APP实现推送。而TLS证书通信，需要为每个APP配置证书。

  • TLS 证书与 APNs 通信

基于令牌的通信有3个步骤：

  • 会员中心获取密钥标识符，得到一个.p8文件和KEY ID
  • 生成JSON身份令牌
  • 发送推送通知

生成身份验证令牌（JWT）

  • // 下面的json数据为JWT的内容, 进行base64URL编码后
// 设置request headers的authorization为bearer <provider token>
{
    "alg": "ES256",                    //加密算法        
    "kid": "ABC123DEFG"            //密钥ID，获取的Key ID
}
{
    "iss": "DEF123GHIJ",        //10位数字的Team ID
    "iat": 1437179036                //生成JWT的UTC时间
 }
