为自己的库创建Pod

发表于 2017-02-10

使用Cocoapods可以很方便的管理第三方类库，我们自己在开发过程中也会总结出自己常用的一些类库。我们也想让我们的类库用Cocoapods管理。

使用Cocoapods可以创建私有的Pod用于公司内容，也可以提交到Cocoapods truck.

  • 创建私有的Pod，用于公司内部。(官方教程)

  • 提交自己项目的podspec文件到CocoaPods trunk。官方提供教程

其实官方给的教程就很全了，没必要看这个文档。

创建项目的podspec文件

不管创建私有的Pod、还是发布到trunk,第一步就是要创建podspec文件。

  • podspec文件的作用
    告诉cocoapods关于类库的基本信息、源码地址、资源内容等。([具体介绍podSpec参数])

  • podspec文件的创建与验证

    • pod lib create : 自动生成一个pod项目所需的所有文件。包括类podSpec文件、DEMO工程。
    • pod lib lint : 验证podSpec文件是否正确。
    • pod spec create your_pod_spec_name : 手动创建podSpec文件

创建私有的Pod

  • 用于公共类库DEMO工程使用

  • 直接指定一个podspec文件、或包含podspec文件的git源

提交podspec文件到 Cocoapods trunk

前提：已经为自己的类库创建了podspec文件，并通过验证。

  • 注册Cocoapods trunk邮箱

  • 发布自己的pods到CocoaPods trunk

  • Cocoapods官方指导
  • 让代码支持CocoaPods

自定义UICollectionView布局的技术要点

发表于 2017-02-10

UICollectionView 构成

  • Cells
  • Supplementary Views 追加视图（类似Header 或则 Footer）
  • Decoration Views 装饰视图（用作背景展示）

布局方法的执行顺序

  1. prepareLayout()
  2. -(CGSize)collectionViewContentSize
  3. -(NSArray *)layoutAttributesForElementsInRect:(CGRect)rect

自定义UICollectionLayout实现自定义布局

主要继承UICollectionLayout以下方法，实现布局自定义。

  • -(void)prepareLayout

    开始布局前，调用的方法。可以用来一次把所有的布局属性放到一个NSArray中。

  • -(CGSize)collectionViewContentSize

    返回CollectionView的内容尺寸

  • -(NSArray *)layoutAttributesForElementsInRect:(CGRect)rect

    1.返回rect中的所有元素的布局属性
    2.返回的是包含UICollectionViewLayoutAttributes的NSArray
    3.UICollectionViewLayoutAttributes可以是Cell、追加视图、装饰视图

    1. 不要返回所有的属性，只显示Rect相交的属性。节省性能。

  • -(BOOL)shouldInvalidateLayoutForBoundsChange:(CGRect)newBounds

    YES : 边界发生变化，刷新布局。
    不要直接返回YES，判断UICollectionView的frame改变时再返回YES。

  • -(UICollectionViewLayoutAttributes _)layoutAttributesForItemAtIndexPath:(NSIndexPath _)indexPath

    返回对应于indexPath的位置的Cell的布局属性

  • -(UICollectionViewLayoutAttributes _)layoutAttributesForSupplementaryViewOfKind:(NSString _)kind atIndexPath:(NSIndexPath *)indexPath

    返回对应于indexPath的位置的追加视图的布局属性

  • -(UICollectionViewLayoutAttributes * )layoutAttributesForDecorationViewOfKind:(NSString_)decorationViewKind atIndexPath:(NSIndexPath _)indexPath

    返回对应于indexPath的位置的装饰视图的布局属性

自定义UICollectionViewLayoutAttributes

主要用来布局cell、追加视图、装饰视图里面的指定子元素。

  • 实现NSCopying协议，配置新增属性的Copy。
  • 实现-(BOOL)isEqual:(id)object父类方法
  • 实现-applyLayoutAttributes:方法，具体布局指定元素。
  • 在UICollectionViewLayout类中实现以下方法
    • +(Class)layoutAttributesClass

重新计算布局

  • invalidateLayout:执行布局方法，从-(void)prepareLayout方法开始重新计算布局。

切换布局

  • setCollectionViewLayout:animated:

自动对齐网格

  • -targetContentOffsetForProposedContentOffset:withScrollingVelocity:

动画效果

  • initialLayoutAttributesForAppearingItemAtIndexPath:
  • initialLayoutAttributesForAppearingSupplementaryElementOfKind:atIndexPath:
  • initialLayoutAttributesForAppearingDecorationElementOfKind:atIndexPath:
  • finalLayoutAttributesForDisappearingItemAtIndexPath:
  • finalLayoutAttributesForDisappearingSupplementaryElementOfKind:atIndexPath:
  • finalLayoutAttributesForDisappearingDecorationElementOfKind:atIndexPath:

  • 猫神的UICollection手册
  • 自定义CollectionView布局 @ObjC中国
  • raywender自定义CollectoionView布局

UICollectionView设置Section的背景色

发表于 2017-02-10

主要使用UICollectionView的装饰视图实现(DecorationView)
技术要点:

  1. 自定义一个UICollectionViewLayoutAttributes,加入color属性用来设置背景色。

    • 在-layoutAttributesForDecorationViewOfKind:方法中，设置新增属性的初始值。

    • 实现NSCopying协议-copyWithZone:,设置新增属性的copy功能。

  2. 实现UICollectionReusableView的子类，用来创建DecorationViewCell

    • 在-applyLayoutAttributes:方法中，设置背景色。

  3. 实现UICollectionViewDelegateFlowLayout子协议。

    • 设置获取Section的背景颜色的委托事件.

  4. 自定义一个UICollectionViewLayout,继承UICollectionViewFlowlayout.

    • 在+layoutAttributesClass返回自定义的UICollectionViewLayoutAttributes.

    • 在-prepareLayout中，注册创建的装饰视图DecorationViewCell.

    • 在-layoutAttributesForElementsInRect:方法，计算并加入当前rect中显示的section的attributes.

      • 将self.collectionView.delegate转换为UICollectionViewDelegateFlowLayout子协议类型。
      • 获取委托事件返回的背景颜色，并设置到自定义的attributes中。
