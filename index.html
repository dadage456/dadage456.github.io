与APNs通信有两种方式

  • 验证令牌与 APNs 通信

    只需要配置一个密钥标识符，可以供多个APP实现推送。而TLS证书通信，需要为每个APP配置证书。

  • TLS 证书与 APNs 通信

基于令牌的通信有3个步骤：

  • 会员中心获取密钥标识符，得到一个.p8文件和KEY ID
  • 生成JSON身份令牌
  • 发送推送通知

生成身份验证令牌（JWT）

  • // 下面的json数据为JWT的内容, 进行base64URL编码后
// 设置request headers的authorization为bearer <provider token>
{
    "alg": "ES256",                    //加密算法        
    "kid": "ABC123DEFG"            //密钥ID，获取的Key ID
}
{
    "iss": "DEF123GHIJ",        //10位数字的Team ID
    "iat": 1437179036                //生成JWT的UTC时间
 }

UIStackView的属性笔记

UIStackView的属性笔记

做iOS开发8年了吧，对于一些技术还是有点模糊。也没有学笔记的习惯，有些问题虽然解决了，但是头脑中没有一个清晰的思路。以至于后来遇到同样的问题，还是需要摸索摸索。

这里主要记录一下：

  • UIStackView的属性的理解

  • UIStackView在 确定了大小约束 和 没有确定大小约束 两种情况下，子视图的布局的不同。

一、UIStackView的基本属性

首先了解一下主轴、纵轴的概念：

主轴: UIStackView的布局方向【布局方向是水平，主轴就是水平】

纵轴：UIStackView的布局的另一个方向 【布局方向是水平，主轴是垂直】

UIStackView主要依靠axis、distribution、alignment、spacing属性进行子视图的布局。

  • axis: 子视图的布局方向，水平和垂直
  • distribution: 子视图的主轴的布局方式
  • alignment：子视图的纵轴的对齐方式
  • spacing：子视图之间的间距

二、distribution属性

对于Fill类型的布局：

  • StackView大小不确定：通过指定子元素大小，确定StackView的大小
  • StackView大小确定：子元素填充StackView的空间

  • UIStackViewDistributionFill

    子元素填满主轴方向StackView空间。

    • StackView大小约束确定：设置其他子元素的高度，留下一个子元素的高度自适应填充StackView空间
  • StackView大小约束不确定： 所有的子元素指定高度

  • UIStackViewDistributionFillEqually

    自动调整子元素在主轴方向的大小，使他们大小一致，填满整个StackView空间。

    • StackView的大小约束确定：子元素自动调整高度。 
* StackView的大小约束不确定：指定一个子元素的高度，其他的子元素自动调整相同高度
  <img src="http://qaan37d6g.bkt.clouddn.com/ios_stackview_distribute_fill_equally_2.png" alt="StackView的大小约束不确定" style="zoom:50%;" />

  • UIStackViewDistributionFillProportionally

    子元素根据自身Frame中的主轴大小，确定占用StackView空间的百分比。

    StackView大小约束确定：子元素会等比例适配

    StackView大小不确定：StackView的大小会根据子元素在主轴上大小，自定适配。

  • UIStackViewDistributionEqualSpacing： StackView的大小约束确定的情况下使用

  • UIStackViewDistributionEqualCentering： **StackView的大小约束确定的情况下使用

ios 的文档管理： 分享、导入、导出保存

#iOS11 读取沙盒以外的文档笔记

[TOC]

一、操作文档的方式

这里看出第二种操作方式，要比第一种简化了一个步骤。

第一种：使用了分享文档，读取其他APP的文档。

第二种：直接打开其他APP的文档。

###二、文件操作的三个步骤

配置info.plist ——> 读取文件——>操作文件

  • 配置info.plist

    • 配置——app可打开的文件的类型
    • 配置——自定义的文件类型

  • 读取文档

    • 应用直接读取沙盒以外的文档
    • 通过其他应用的调用系统的分享组件，分享文档

  • 操作文档： 创建、打开、保存等操作

###三、直接读取沙盒以外的文档

  • UIDocumentPickerViewController

    指定操作文件的类型， 操作的方式：打开、导入、导出、移动

  • UIDocumentBrowserViewController

    创建基于文档视图的App, XCode ——> File ——> Project——> Document Based App

    使用回调来完成文档的创建、文档的选择操作。

    • 添加自定义的动作：设置customActions属性，在长按弹出菜单、导航栏菜单功能项增加菜单

    • 执行回调方法，在UIActivityController分享栏添加自定义动作

    • 使用UIDocumentBrowserTransitionController实现动画过渡

  • 对读取文档的操作

    • 使用NSFileCoordinator访问选中的文档

    • 使用UIDocument对文档进行创建、打开、保存、重命名、定时保存等操作。

四、如何让应用在Files中显示Document文档中的内容

  • 应用如何显示在Files的设置

    • 在info.plist文件设置

      • Application supports iTunes file sharing: true

        在iTunes中显示应用的Documents文件夹

      • Supports opening documents in place: true

        在Files中显示该应用的Documents文件夹

    • 针对与基于文档浏览器UIDocumentBrowserViewController，在info.plish文件中设置

      • Supports Document Browser: true

        Files中是否可以直接点击文件，使用该应用打开。

  • 在iOS的设置——》隐私——》文件和文件夹, 设置当前应用允许显示。

  • 如果要隐藏Documents文档中的文档，文件名前加上.

  • 设置自定义文件的缩略图： 缩略图扩展 Thumbnail Extension
  • 设置自定义文件的快速预览： 快速预览扩展 Quick Look Preview Extension
